---
title: Variables
---

Environment variables let you configure how your application behaves in different environments. You can add, edit, and
delete environment variables as needed.

You can access the environment variables by navigating to `Application > Environment > Variables`.

<img src="/img/environment-variables.png" alt="Environment Variables" className="image" />

### App Environment Variables

Variables extracted from app source code and value can't be changed. Example: `DatabaseUrl`, `S3_ENDPOINT` etc.

### Framework Environment Variables

Variables defined by Low-Ops that follow best practices of specific frameworks. Example: `Runtime settings` and
`Log levels` for Mendix Applications.

### Custom Environment Variables

Environment variables defined by the user. You can add, edit and delete any variables that you need in your application.
