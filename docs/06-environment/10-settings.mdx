Configure your environment with settings for domain, scaling, and environment management.

Go to `Application > Environment > Settings` to access these settings.

### Domain Settings

Domain settings configure your environment's domain. The default URL format is `[app name]-[environment name].[domain]`.
You can:

- Use a custom URL
- Add `TLS Certificate` and `TLS Certificate Key` to enable TLS

<img src="/img/domain-settings.png" alt="Domain Settings" className="image" />

:::warning

Re-deploy the environment to apply new domain settings. <ReDeployLink />

:::

### Scaling Settings

Adjust your environment's resources:

- CPU allocation
- Memory allocation
- Number of replicas

<img src="/img/scaling-settings.png" alt="Scaling Settings" className="image" />

:::warning

Re-deploy the environment to apply new scaling settings. <ReDeployLink />

:::

### Delete Environment

Remove the current environment and all its resources.

<img src="/img/delete-environment.png" alt="Delete Environment" className="image" />

:::warning

This action cannot be undone. All environment resources will be permanently deleted.

:::

### Mendix Access Settings

Get login credentials for:

- Mendix Studio Pro
- Mendix Debugger

<img src="/img/mendix-access.png" alt="Mendix Access" className="image" />

### Mendix Runtime Settings

Manage Mendix runtime configuration:

- Add new settings
- Edit existing settings
- Remove settings

<img src="/img/mendix-runtime.png" alt="Mendix Runtime" className="image" />

:::note

To override the application URL, add `ApplicationRootUrl` with your custom URL as the `VALUE`.

:::

:::warning

Re-deploy the environment to apply new runtime settings. <ReDeployLink />

:::
